Why Virtual Environments?

A virtual environment is an isolated Python installation that lets you install packages without affecting the system Python or other users.

Problems they solve

  • Version conflicts: Project A needs numpy 1.21, Project B needs numpy 1.26
  • Permission issues: You can't install system-wide packages on the cluster
  • Reproducibility: Pin exact versions for consistent results
  • Isolation: Break one environment without affecting others
On ilifu: Always use a virtual environment for your project code. The shared cluster environment is not yours to modify, and other users depend on its stability.

Step 1 — Choose a Python Version

The compute nodes run Ubuntu 20.04, providing Python 3.8 by default. Use module to access other versions.

List available Python modules

module avail

Look for entries like python/3.9.4, python/3.10.x, etc.

Load a specific Python version

module load python/3.9.4

Verify the version

python --version
# Python 3.9.4
Note: If Python 3.8 is sufficient for your work, you do not need to load any module — it is available by default. Only load a module if you need a specific version.

Step 2 — Create & Activate the Environment

Allocate a development node first

Never create environments or install packages on the login node. Get a compute node first:

sinteractive

Your prompt changes to show the compute node, e.g. username@compute-101:~$

Create the virtual environment

virtualenv .venv/tutenv

This creates a new Python environment at .venv/tutenv/

Activate the environment

source .venv/tutenv/bin/activate

Your prompt now shows the active environment:

(.tutenv) username@compute-101:~$
Key point: Activating updates $PATH so that python and pip now refer to the virtual environment's versions. Always activate before installing packages or running code.

Step 3 — Install Packages

With the virtual environment active, use pip to install packages. They will only be available inside this environment.

pip install scikit-learn
pip install numpy pandas matplotlib

Verify installation

python -c "import sklearn; print(sklearn.__version__)"

Deactivate when done

deactivate

Best practices

  • Always activate before pip install
  • Create one environment per project
  • Use pip freeze > requirements.txt to record dependencies for reproducibility
  • Do not use sudo pip install (you can't on the cluster anyway)
  • Don't mix environments between projects

Step 4 — Register as a Jupyter Kernel

To use your virtual environment in Jupyter notebooks, register it as a kernel.

Install the kernel package

pip install ipykernel

Register the kernel

ipython kernel install --name "tutenv_py3.9.4" --user

This creates a kernel config at:

~/.local/share/jupyter/kernels/tutenv_py3.9.4/kernel.json

Use the kernel in Jupyter

  • New notebook: Select your kernel from the launcher dropdown
  • Existing notebook: Kernel menu → Change Kernel → select your environment
Result: Any package installed in .venv/tutenv is now available in Jupyter notebooks that use this kernel — without modifying the shared cluster environment.

Quick Reference — Complete Workflow

# 1. Load desired Python version (optional)
module load python/3.9.4

# 2. Get a compute node
sinteractive

# 3. Create virtual environment
virtualenv .venv/tutenv

# 4. Activate it
source .venv/tutenv/bin/activate

# 5. Install your packages
pip install scikit-learn numpy pandas ipykernel

# 6. Register as Jupyter kernel
ipython kernel install --name "tutenv_py3.9.4" --user

# 7. Deactivate when done
deactivate
