What Are Job Arrays?

This tutorial shows an example of how to submit a Slurm array job.

A Slurm array job is a way for submitting and managing collections of similar jobs quickly and easily. Each job in the array gets a different parameter as input, making sure that they run different computations.

Use cases

  • Processing 100 input files — one job per file
  • Running the same analysis with many different parameters
  • Cross-validation folds in machine learning
  • Bootstrap resampling across multiple seeds
Efficiency: Job arrays are much easier to manage than submitting many individual jobs — and SLURM tracks them all under a single array job ID.

Tutorial Files

There are two files in this tutorial:

  • myscript.py — a dummy Python script that prints the index it received
  • slurm-arrayjob.sh — the sbatch script defining how the job array will work

In this case the Slurm array job will be created with 20 jobs, where 5 are run concurrently. The indexes of these jobs will be from 1 to 20. The dummy script receives these indexes as input and just prints out which index it received.

Testing the Script Manually

You can try out the dummy script yourself by calling it with a random number (e.g. with 10):

module load python3
python3 myscript.py --input 10

You should see the following output in the terminal:

The input parameter is: 10

The Array Job sbatch Script

The slurm-arrayjob.sh script submits 20 tasks (indexes 1–20) with at most 5 running concurrently:

#!/bin/bash
#SBATCH --job-name=myarrayjob
#SBATCH --array=1-20%5          # Indices 1–20, max 5 concurrent
#SBATCH --output=logs/myarrayjob-%A_%a.out   # %A = job ID, %a = array index
#SBATCH --error=logs/myarrayjob-%A_%a.err
#SBATCH --time=00:05:00
#SBATCH --cpus-per-task=1
#SBATCH --mem=1G

module load python3

# $SLURM_ARRAY_TASK_ID is the current array index (1, 2, 3, … 20)
python3 myscript.py --input $SLURM_ARRAY_TASK_ID

Key directives

  • --array=1-20%5 — 20 tasks with indexes 1–20; the %5 throttle means at most 5 run simultaneously
  • %A — the job array ID (same for all tasks)
  • %a — the individual task index
  • $SLURM_ARRAY_TASK_ID — the current task's index at runtime

Submitting the Array Job

Next we are going to run the array job that calls the script 20 times, but with only 5 running concurrently. Run the following command:

sbatch slurm-arrayjob.sh

After submitting the job, you should see a job ID printed in the terminal.

Monitoring the Array

You will also be able to see the job with that job ID in the queue:

squeue -u $USER

In order to see how the jobs get completed and how only 5 are run concurrently, you can use the following command. This will refresh the squeue output every 2 seconds:

watch "squeue -u $USER"

You'll see tasks appear as 1000_1, 1000_2 … 1000_20 (where 1000 is your job ID), and see them move from PENDING → RUNNING → disappear as they complete. At most 5 run simultaneously.

Checking Output

After the array job completes, you can look at the output generated in the logs folder. There should be a .out file with the standard output for every one of the 20 array jobs. These will be similar to the output you received by running the script manually.

For example, if your job ID was 1000, the output for the 5th job will be:

cat logs/myarrayjob-1000_5.out
# The input parameter is: 5

There will also be .err files for the standard error: myarrayjob-1000_5.err. This file should be blank if everything went well.

Array jobs are especially useful for automating the running of the same scripts with different inputs. They replace the need to write and submit dozens of individual sbatch scripts.
