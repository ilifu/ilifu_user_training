What is Jupyter on ilifu?

ilifu provides JupyterHub — a multi-user Jupyter environment that runs your notebooks directly on the HPC cluster. You access it through your browser; the notebook kernel runs on a compute node allocated by SLURM.

  • Access via the ilifu portal — no local installation needed
  • Your notebook runs on a compute node allocated by SLURM
  • Full access to cluster storage, modules, and compute resources
  • Supports Python, R, and custom virtual environment kernels
Why use Jupyter on the cluster? Large datasets that won't fit on your laptop, access to powerful GPUs and many-core CPUs, and shared project storage — all in a familiar notebook interface.

Architecture

When you start a session on JupyterHub, SLURM allocates a compute node and the Jupyter kernel runs there — not on the login node.

Your Browser → JupyterHub portal → SLURM Scheduler → Compute Node
                                                        (Jupyter kernel runs here)

Accessing JupyterHub on ilifu

Step 1 — Log in to the portal

  • Navigate to the ilifu JupyterHub URL (provided by your instructor)
  • Sign in with your ilifu credentials

Step 2 — Select a server profile

  • Choose CPUs, memory, and runtime (SLURM will allocate accordingly)
  • Click Start — your session is queued as a SLURM job
  • Wait for the server to start; this may take a minute while SLURM queues and schedules the job

Step 3 — Open the tutorial notebook

  • Navigate to session1/tutorial1/ in the file browser
  • Open the .ipynb file
  • Select the appropriate kernel (default Python is fine for Tutorial 1)

Keyboard Shortcuts

Jupyter has two modes: Command mode (press Esc) for cell-level operations, and Edit mode (press Enter) for typing in a cell.

ShortcutModeAction
Shift+EnterEitherRun cell and advance to next
Ctrl+EnterEitherRun cell and stay
ACommandInsert cell above
BCommandInsert cell below
D, DCommandDelete cell
MCommandChange cell to Markdown
YCommandChange cell to Code
Ctrl+SEitherSave notebook
Tip: Press Esc first to enter command mode, then use single-key shortcuts. Press Enter to return to edit mode.

Key Features Demonstrated in the Notebook

Working with Data

  • Loading files from cluster storage paths
  • Using pandas for tabular data
  • Plotting with matplotlib

Shell Commands in Notebooks

Prefix any shell command with ! to run it from a notebook cell:

!ls                    # list files
!squeue -u $USER       # check your SLURM jobs
!cat data/sample.txt   # print file contents

Mix Python and shell output in the same notebook for interactive exploration.

Markdown Cells

  • Document your analysis with formatted text
  • Headers (# Title, ## Section), bullet points, bold/italic
  • Equations using LaTeX: $E = mc^2$

Saving & Checkpoints

  • Notebooks auto-save periodically
  • Manual save: Ctrl+S or File → Save
  • Checkpoints allow reverting to a previous state via File → Revert to Checkpoint
Custom kernels: After Session 2 Tutorial 1, you'll be able to register your own Python virtual environments as Jupyter kernels — making your packages available in notebooks without modifying the shared cluster environment.
